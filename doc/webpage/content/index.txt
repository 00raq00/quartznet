<h1>Welcome to the home of Quartz Enterprise Scheduler .NET!</h1>
<br />
<cite>Quartz.NET is a full-featured, open source job scheduling system that can be used from smallest apps to large scale enterprise systems.</cite>

<p>
	Quartz.NET is a port of very propular open source Java job scheduling framework, <a href="http://www.opensymphony.com/quartz">Quartz</a>. 
	This project owes very much to original Java project, it's father James House and
	the project contributors. 
</p>
<p>
	Quartz.NET is a pure .NET library written in C# which currently targets Framework version 1.1 and above.
	Quart.NET is still very much work in progress, all feedback and suggestions are greatly welcome.
</p>

<h1>Quartz.NET 0.6 Released</h1>
<div class="descr">August 4, 2007 by Marko Lahma</div>
<p>
	Quartz.NET 0.6 has been released. 
</p>
<p>
	This both feature and bug fix release. The new biggest feature is AdoJobStore
	that currently supports SQL Server, Oracle, MySQL, PostgreSQL, SQLite.
	With AdoJobStore comes also clustering. Please give it a go (Example 13 has an example 
	for SQL Server) and give us feedback!
</p>
<p>
	This release also fixes many problems found by users, most notably CronExpression problems
	when having a more frequent firing schedule, like every nth second or minute.
</p>
<p>
	Now minimal configuration is also automatically loaded from Quartz.NET embedded resource, 
	so if you don't want to change the default configuration, you don't need to type anything anymore.
	See <a href="http://jira.opensymphony.com/secure/ReleaseNote.jspa?projectId=10110&styleName=Html&version=21710">
	release notes</a> for more details.
</p>
<p>
	You should be also aware of some <strong>breaking changes</strong>, please check 
	changelog.txt in the distribution root.
</p>
<p>
	Grap it from <a href="download.html">downloads page</a>.
</p>
